============================================================
GIT CHEATSHEET — DustCollector (v2-architecture)
============================================================

ASSUMPTIONS
-----------
• You are inside the repo directory
• You want boring, safe, explicit commands
• No rebases, no force-pushes, no heroics


------------------------------------------------------------
1. CHECK WHERE YOU ARE
------------------------------------------------------------

# Show current branch
git branch

# Show status (what changed, what’s staged)
git status

# Show recent commits (short)
git log --oneline --decorate -5


------------------------------------------------------------
2. ADD CHANGES, COMMIT, PUSH (v2-architecture)
------------------------------------------------------------

# Make sure you are on the right branch
git checkout v2-architecture

# See what changed
git status

# Add specific files
git add v2/main.py v2/tasks v2/domain

# OR add everything you touched (be careful, but common)
git add .

# Commit with a clear message
git commit -m "Add SSR control; preserve ADC-driven gate behavior"

# Push to origin
git push origin v2-architecture


------------------------------------------------------------
3. PULL DOWN LATEST v2-architecture CHANGES
------------------------------------------------------------

# Switch to the branch (if not already there)
git checkout v2-architecture

# Pull latest commits from remote
git pull origin v2-architecture


------------------------------------------------------------
4. SEE WHAT CHANGED (WITHOUT EDITING)
------------------------------------------------------------

# Show unstaged changes
git diff

# Show staged changes
git diff --staged

# Show changes from last commit
git show


------------------------------------------------------------
5. DISCARD LOCAL CHANGES (USE WITH CARE)
------------------------------------------------------------

# Discard changes to a single file
git checkout -- v2/main.py

# Discard ALL local changes (DANGEROUS)
git checkout -- .


------------------------------------------------------------
6. SAVE WORK TEMPORARILY (STASH)
------------------------------------------------------------

# Stash current work
git stash

# See stashes
git stash list

# Reapply most recent stash
git stash pop


------------------------------------------------------------
7. UPDATE LOCAL BRANCH FROM REMOTE (SAFE WAY)
------------------------------------------------------------

# Fetch everything (no changes yet)
git fetch origin

# Compare local vs remote
git log --oneline --decorate --graph HEAD..origin/v2-architecture

# Then pull
git pull origin v2-architecture


------------------------------------------------------------
8. CREATE A ZIP SNAPSHOT (FOR BACKUP / SHARING)
------------------------------------------------------------

# From repo root
git archive --format=zip \
    --output=dustcollector_v2_snapshot_$(date +%Y%m%d_%H%M).zip \
    v2-architecture


------------------------------------------------------------
9. IF SOMETHING FEELS WRONG
------------------------------------------------------------

# Show remote URLs
git remote -v

# Verify branch tracking
git branch -vv

# Abort a merge in progress
git merge --abort


------------------------------------------------------------
10. RULES OF THUMB (PRINT THESE)
------------------------------------------------------------

• git status before anything
• git add deliberately
• commit often, push when stable
• never force-push unless you *really* mean it
• when confused: STOP, git status, git branch, git log
